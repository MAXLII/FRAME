# 上位机主页 UI 重设计说明（给 Codex 继续设计）

## 1. 设计目标
基于当前首页截图与《上位机》文档第 7 章“主页”需求，对主页进行增删改设计：
- 保留第 7 章要求的核心业务信息
- 删除当前页面中与主页弱相关或占位过大的内容
- 强化“能源流向 + 设备状态 + 异常信息 + 关键控制”的首页属性
- 产出一份可直接给 Codex 继续做界面实现/视觉深化的说明文件

---

## 2. 设计输入来源

#### 市电
- AC电压有效值
- AC电流有效值

- 频率

#### 逆变

## 3. 现有页面问题（结合截图）
当前页面已经有“太阳能/电池/并网/负载”四张统计卡片，也有中部能量示意图，但存在以下问题：

1. **首页与第 7 章内容不完全对齐**
   - 现有页面更偏“电站统计首页”
   - 第 7 章要求的是“PCS/逆变电源设备主页”，强调实时运行与控制

2. **中部示意图信息密度不足**
   - 图标分散
   - 能量流向关系不够清晰
   - 没有把市电 / 逆变 / 电池 / MPPT / 负载关系讲清楚

3. **天气区域占位偏大**
   - 与主页核心监控关系较弱
   - 可缩小为辅助卡片或移出首页

4. **状态/告警信息不突出**
   - 第 7 章要求保护、故障、告警必须明确显示
   - 当前首页缺少集中告警区

5. **缺少直接控制入口**
   - 第 7 章需要 AC 输出开关和电压频率设置
   - 当前首页没有明显控制区

## 4. 首页重设计原则
### 保留
- 顶部导航
- 站点/设备选择
- 四张总览卡片的“统计概览”思路
- 首页的能源监控属性

### 删除/弱化
- 过大的天气说明区
- 过多留白
- 不成体系的零散设备图标摆放

### 新增
- 中心能量流图
- 状态灯区
- 保护/故障/告警独立卡片
- AC 输出控制区
- 关键实时参数区
- 趋势小图或实时曲线入口

---

## 5. 推荐新页面结构

内容：
- 页面标题：`主页 / Dashboard`
- 站点选择
- 设备地址 / 模块地址选择
- 连接状态（在线/离线）
- 最后刷新时间

说明：
- “在线/离线”必须是明显状态标签，不要只写文字
- 建议增加串口/通信状态小图标

## 5.3 第二行左侧：中心能量流图（主页核心）
用一张大的能量流图替代现在松散的系统示意区域。

### 布局建议
- 左上：MPPT / PV
- 左下：电池
- 中间：逆变器 PCS
- 右上：市电 Grid
- 右下：负载 Load
- 底部或右下角：EPS

### 每个节点要显示的数据
#### 市电 Grid
- AC电压
- AC电流
- 功率
- 频率

#### 逆变 PCS
- 输出电压
- 输出频率
- 当前目标电压
- 当前目标频率
- 运行状态（待机 / 运行 / 故障）

#### 电池 Battery
- 电压
- 电流
- SOC
- 功率
- 温度

#### MPPT / PV
- 输入电压
- 输入电流
- 输入功率
- 光伏状态灯

#### 负载 Load
- 当前输出功率

#### EPS
- EPS状态灯

### 连线关系
- PV -> PCS
- Battery <-> PCS
- Grid <-> PCS
- PCS -> Load
- PCS -> EPS

### 交互建议
- 有功率流动时连线高亮
- 充电/放电方向用箭头表示
- 离线时节点灰化
- 故障时 PCS 节点红色高亮

---

## 5.4 第二行右侧：状态与异常面板
分成三块小卡片，优先级从上到下排列：

### 运行状态
- 风扇状态
- 继电器状态
- 光伏板状态
- EPS状态
- 功率管温度
- INFO文本

### 保护状态（可恢复）
- 列表文本显示
- 黄色风格
- 可显示多条

### 故障状态（不可恢复）
- 列表文本显示
- 红色风格
- 高优先级置顶

### 告警状态
- 列表文本显示
- 橙色或蓝黄色风格
- 单独区域

说明：
- 保护 / 故障 / 告警必须区分颜色和图标
- 不可混成一个“消息框”

---

## 5.5 第三行：控制与设置区
放在首页底部右侧或异常区下方。

### 控制内容
- AC 输出开关按钮
- 电压频率组合单选
  - 220V / 50Hz
  - 230V / 50Hz
  - 240V / 50Hz
- 设置按钮

### 交互规则
- 当前生效项高亮
- 未下发配置时显示“待应用”
- 设置成功/失败有 toast 和状态提示

以下内容不建议继续占据首页核心位置：

1. **大面积天气卡片**
   - 改成右侧小卡片
   - 或放到二级页

2. **累计发电/耗电大卡**
   - 可保留在统计页
   - 首页优先实时量

3. **过多空白占位图标**
   - 要改成真实设备关系图
   - 不能只是装饰图标

## 9. 前端实现建议（给 Codex）
### 页面建议拆分组件
- `DashboardHeader`
- `OverviewCard`
- `EnergyFlowPanel`
- `StatusPanel`
- `FaultPanel`
- `WarningPanel`
- `ControlPanel`
- `RealtimeTable`
- `TrendPanel`

### 状态枚举建议
- 在线状态：`online / offline / warning / fault`
- 风扇状态：`on / off`
- 继电器状态：`open / close`
- PV状态：`detected / not_detected`
- EPS状态：`available / unavailable`

### 数据刷新
- 首页实时数据建议 500ms ~ 1000ms 刷新
- 状态/故障列表支持去抖与变化高亮

### 异常优先级
- Fault > Protect > Warning > Info

## 10. 数据映射建议（按广播结构）
主页数据建议直接从 `pcs_info_t` 绑定：

- `mppt_vin` -> MPPT输入电压
- `mppt_iin` -> MPPT输入电流
- `mppt_pwr` -> MPPT输入功率
- `ac_v_grid` -> 市电电压
- `ac_i_grid` -> 市电电流
- `ac_freq_grid` -> 市电频率
- `ac_pwr_grid` -> 市电功率
- `ac_v_inv` -> 逆变输出电压
- `ac_i_inv` -> 逆变输出电流
- `ac_pwr_inv` -> 逆变输出功率
- `ac_freq_inv` -> 逆变输出频率
- `pfc_temp / llc_temp1 / llc_temp2` -> 功率管温度
- `fan_sta` -> 风扇状态
- `rly_sta` -> 继电器/MOS状态
- `protect` -> 保护状态
- `fault` -> 故障状态
- `warning` -> 告警状态
- `bat_volt` -> 电池电压
- `bat_curr` -> 电池电流
- `bat_pwr` -> 电池功率
- `bat_temp` -> 电池温度
- `soc` -> SOC

设置区数据来自 `inv_cfg_t`：
- `ac_out_enable_trig`
- `ac_out_disable_trig`
- `ac_out_rms`
- `ac_out_freq`
